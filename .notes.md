## Creating tables in your project with prisma schema

prisma is the language and schema is just structure
To create a table
define a model and the name of the table starting with a Capital letter
The rows of the table is defined, with the type of input it requires and checks for example `id`, `default` or `unique`.

**This is to be done everytime we create a table or make changes to the table. Like a git push changes**
After creating your table.

- Make sure the database is configured in prisma config file.
- And the schema client is upto date
  `yarn prisma migrate dev --name add_users_table`

- Always have prisma generate the types after migration
  `yarn prisma generate` - This requires `@prisma/client` thus installation required.

For production deployment - `yarn prisma migrate deploy`

`yarn prisma studio` - gives you a hosting to see your work

- Configure the prisma.config.ts to be able to export the adapter into the db.js to be able to migrate it.
- This is followed in the docs

## Establishing relationships across the movies

- With this we are creating a connection between the tables showing that a field in one table should reflect another field in the relating table - userId in watchList table to id in User.
- We can also include a nifty feature to handle deletion in the of user and relation to the watchList.

```prisma

user User @relation(fields:[userId], references: [id], onDelete: Cascade)
```

This error means that your User model does not have a corresponding relation field for the WatchListItem model. In Prisma, relations must be defined on both sides.
